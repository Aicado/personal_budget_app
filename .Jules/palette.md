## 2026-05-04 - [Empty State Recovery Action]
**Learning:** When a user applies a filter that results in an empty state (e.g., "Needs Data" filter when all accounts are complete), the interface can feel like a dead-end. Providing an immediate, contextual recovery action (like a "Show All" button) within the empty state message significantly improves navigation and user confidence.
**Action:** Always include a primary recovery action in empty states that result from user-initiated filtering or searching.
